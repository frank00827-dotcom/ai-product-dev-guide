## 经验总结

### AI 辅助开发最佳实践

1. **明确声明技术栈** - Prompt 开头指定 Flask/SQLAlchemy 版本
2. **提供上下文** - 粘贴相关模型和已有代码
3. **分步生成** - 先模型→再 Service→最后 Routes
4. **及时审查** - 每轮生成后检查安全、性能、规范
5. **记录决策** - 开发日志记录关键决策和原因

### Prompt 工程要点

**好的 Prompt**:
```markdown
# 角色
你是 Flask 后端开发专家。

# 技术栈
Flask 3.x + SQLAlchemy 2.x + PostgreSQL

# 需求
[清晰的功能描述]

# 约束
- 安全要求（如：SQL 注入防护）
- 性能要求（如：响应时间<200ms）
- 代码规范（如：PEP8）

# 输出格式
按文件分别输出，标注路径
```

**避免的陷阱**:
- ❌ 技术栈混用（NestJS + Flask）
- ❌ 需求模糊（"实现用户管理"）
- ❌ 缺少约束（无安全、性能要求）
- ❌ 一次性生成整个系统
