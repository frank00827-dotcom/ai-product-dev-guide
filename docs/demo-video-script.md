### 【1:30-3:00】快速跑通演示

**画面**: 切换到终端，演示快速启动

**台词**:
```
现在我来演示如何快速跑通这个项目。

第一步，启动基础设施：
```bash
docker compose -f deploy/docker-compose.yml up -d
```

这会启动 PostgreSQL 数据库、MinIO 对象存储和 Redis 缓存。

第二步，初始化后端：
```bash
cd server
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
flask db upgrade
flask run
```

后端启动成功！访问 http://localhost:5000/health 可以看到健康检查通过。

第三步，启动前端：
```bash
cd client
npm install
npm run dev:weapp
```

然后在微信开发者工具中导入项目，勾选"不校验合法域名"。

搞定！现在你可以在小程序里完成登录、录入健康数据等操作。
```

**画面提示**:
- 终端命令逐行高亮
- docker compose ps 显示 healthy 状态
- 微信开发者工具界面

### 【4:30-5:30】技术栈与架构

**画面**: 切换到架构图（C4 模型或自定义架构图）

**台词**:

前端采用 Taro 3.x，这是一个跨端框架，可以同时编译到微信小程序、支付宝小程序和 H5。

状态管理使用 Zustand，这是一个轻量级的方案，比 Redux 更简洁。

后端采用 Flask 模块化架构，每个业务模块独立管理自己的路由、服务和数据模型。

数据库使用 PostgreSQL 16，配合 Alembic 做迁移管理。

对象存储使用 MinIO，可以无缝切换到阿里云 OSS 或 AWS S3。

测试方面，后端使用 pytest，前端使用 Jest + React Testing Library，总体覆盖率 86%。
