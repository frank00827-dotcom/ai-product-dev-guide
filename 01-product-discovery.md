# 01 - 产品发现与PRD

## 1.1 AI 辅助市场调研

### 方法论

产品发现阶段的核心目标是**验证问题的真实性和解决方案的可行性**。AI 在此阶段的价值是快速聚合信息、生成结构化分析，但市场判断的最终决策权在人。

### 工作流

```
┌─────────────┐    ┌──────────────┐    ┌──────────────┐    ┌─────────────┐
│ 行业背景研究 │───→│ 竞品分析     │───→│ 用户画像定义 │───→│ 需求提炼    │
│ (AI Research)│    │ (AI + 人工)  │    │ (AI Draft)   │    │ (人工主导)  │
└─────────────┘    └──────────────┘    └──────────────┘    └─────────────┘
```

### Prompt 模板：行业背景研究

```markdown
# 角色
你是一名资深医疗健康行业分析师，擅长慢病管理和数字疗法领域。

# 任务
请对中国慢病管理市场进行深度分析，输出结构化报告，包含：

1. **市场规模与趋势**
   - 中国慢病患者基数（糖尿病、高血压、COPD）
   - 数字疗法市场规模及增长预测
   - 政策驱动因素（医保、分级诊疗、互联网医疗）

2. **竞品格局**
   - 头部产品（至少5个）的核心功能矩阵
   - 各产品的差异化定位
   - 未被满足的用户需求（Gap Analysis）

3. **技术趋势**
   - AI 在慢病管理中的应用场景
   - 可穿戴设备数据整合现状
   - 数字疗法的监管政策演进

# 输出格式
使用 Markdown 表格和结构化列表，数据尽量标注来源。
```

### Prompt 模板：竞品分析

```markdown
# 角色
你是一名产品分析专家。

# 任务
对以下慢病管理产品进行功能对比分析：
- 智云健康
- 微医
- 丁香医生（慢病管理模块）
- 医联
- 百度健康

# 分析维度
| 维度 | 说明 |
|------|------|
| 核心功能 | 随访、用药提醒、健康数据、医患沟通、数字疗法 |
| 用户体验 | 注册流程、核心路径步数、信息架构 |
| 技术实现 | 平台（App/小程序/H5）、数据采集方式 |
| 商业模式 | 付费方式、B端/C端/G端 |
| 合规能力 | 医疗器械注册证、等保认证 |

# 输出
1. 功能对比矩阵（表格）
2. 各产品 SWOT 分析
3. 市场空白点总结
```

## 1.2 用户画像定义

### AI 辅助生成 + 人工校准

```markdown
# Prompt：生成用户画像

基于慢病随访与数字疗法场景，生成以下三类核心用户画像：

## 要求
每个画像包含：
- 基本人口统计信息
- 一天的典型场景描述（Scenario）
- 痛点列表（Pain Points）按严重程度排序
- 期望的解决方案（Desired Outcomes）
- 技术使用习惯和能力
- 关键行为指标（使用频率、决策因素）

## 用户类型
1. **患者用户** — 50-70岁慢病患者（以2型糖尿病为主要画像）
2. **医护用户** — 社区卫生服务中心全科医生
3. **健康管理师** — 第三方健康管理机构从业者

## 约束
- 患者画像需考虑数字素养较低的情况
- 医护画像需考虑工作负荷和现有系统兼容性
- 所有画像基于中国医疗体系背景
```

### 案例产出：患者用户画像

> 以下为 AI 生成初稿，经产品团队实地调研校准后的版本

**张阿姨 — 2型糖尿病患者**

| 属性 | 描述 |
|------|------|
| 年龄 | 62岁 |
| 职业 | 退休教师 |
| 病程 | 确诊8年，口服降糖药+胰岛素 |
| 设备 | 华为手机，微信重度用户，不习惯下载新App |
| 数字素养 | 会用微信聊天、朋友圈、小程序买菜，不会复杂操作 |

**典型一天**：
- 06:30 起床测空腹血糖，手写记录在本子上
- 07:00 早餐前注射胰岛素，有时忘记
- 10:00 社区医院复诊（每月一次），排队等候，医生看3分钟
- 12:00 午餐，不确定哪些食物该少吃
- 20:00 忘记测餐后血糖

**痛点排序**：
1. 🔴 血糖记录分散，复诊时医生看不到完整数据
2. 🔴 用药提醒不及时，经常漏服
3. 🟡 不知道日常饮食和运动如何影响血糖
4. 🟡 两次复诊之间出现异常不知道该不该去医院
5. 🟢 想了解病情趋势但看不懂检查报告

## 1.3 PRD 生成

### AI 辅助 PRD 工作流

```
用户画像 + 竞品分析 ──→ AI 生成 PRD 初稿 ──→ 产品经理审查修订
                                                    │
功能优先级排序 ←──── 团队评审 ←──── 补充业务规则 ←──┘
```

### Prompt 模板：PRD 生成

```markdown
# 角色
你是一名资深产品经理，擅长医疗健康产品设计。

# 上下文
我们正在开发一款微信小程序「慢病管家」，目标用户包括慢病患者、
基层医护人员和健康管理师。请参考以下用户画像和竞品分析结果：
[粘贴前序阶段的产出物]

# 任务
生成完整的 PRD 文档，包含以下章节：

## 必须包含
1. 产品概述（愿景、目标、成功指标）
2. 功能需求（按模块组织，每个功能包含用户故事、验收标准）
3. 非功能需求（性能、安全、可用性）
4. 功能优先级矩阵（MoSCoW 方法）
5. 信息架构（页面层级和导航结构）
6. 核心用户流程（至少3个关键流程的步骤描述）

## 格式要求
- 每个功能需求使用用户故事格式：As a [user], I want [goal], so that [benefit]
- 验收标准使用 Given-When-Then 格式
- 优先级使用 Must/Should/Could/Won't 标注
```

### 案例产出：功能优先级矩阵（MoSCoW）

| 优先级 | 模块 | 功能 | 用户故事 |
|--------|------|------|----------|
| **Must** | 患者档案 | 基本信息录入 | 作为患者，我想快速完成建档，以便开始使用服务 |
| **Must** | 患者档案 | 微信授权登录 | 作为患者，我想用微信直接登录，不用记新密码 |
| **Must** | 健康数据 | 血糖手动录入 | 作为患者，我想记录每次血糖值，以便医生查看趋势 |
| **Must** | 健康数据 | 血压手动录入 | 作为患者，我想记录血压值，追踪控制情况 |
| **Must** | 随访管理 | 随访计划创建 | 作为医生，我想为患者创建随访计划，确保按时跟进 |
| **Must** | 随访管理 | 随访提醒推送 | 作为患者，我想收到随访提醒，不错过复诊时间 |
| **Must** | 用药管理 | 用药提醒 | 作为患者，我想定时收到吃药提醒，避免漏服 |
| **Should** | 健康数据 | 蓝牙设备数据同步 | 作为患者，我想自动同步血糖仪数据，免去手动录入 |
| **Should** | 数字疗法 | 健康教育内容 | 作为患者，我想学习疾病知识，更好地自我管理 |
| **Should** | 数字疗法 | 饮食方案推荐 | 作为患者，我想知道每餐该怎么吃 |
| **Should** | 数字疗法 | 运动方案推荐 | 作为患者，我想获得适合我的运动建议 |
| **Should** | 随访管理 | 随访问卷模板 | 作为医生，我想使用标准化问卷高效完成随访 |
| **Should** | 数据看板 | 患者群组管理 | 作为医生，我想分组管理患者，聚焦高危人群 |
| **Could** | 健康数据 | AI 趋势分析 | 作为患者，我想看到血糖趋势和预测 |
| **Could** | 数字疗法 | 个性化干预方案 | 作为患者，我想获得基于我数据的个性化建议 |
| **Could** | 医患沟通 | 图文咨询 | 作为患者，我想在线咨询医生简单问题 |
| **Won't** (v1) | 社交 | 病友社区 | 留待后续版本 |
| **Won't** (v1) | 电商 | 药品购买 | 涉及药品资质，留待后续版本 |

## 1.4 阶段交付物检查清单

- [ ] 行业背景研究报告（AI 生成 + 人工标注来源）

- [ ] 用户画像 × 3（AI 初稿 + 用户访谈/调研校准）
- [ ] 完整 PRD 文档（含功能需求、非功能需求、优先级）
- [ ] 功能优先级矩阵（团队评审确认）
- [ ] 信息架构图（页面层级导航）
- [ ] 核心用户流程图（至少 3 条关键路径）
